# Пошаговая инструкция для студентов

## Стенд UAV Lon ESP8266 — запуск, настройка, фильтры и измерения

---

## 1. Цель стенда

Квадрокоптер закреплён на **неподвижной оси** (режим **1 степень свободы** — в основном **pitch**).

Задачи лабораторной работы:

1. Запустить наземную станцию Qt и связь с бортом (ESP32 через ESP8266).
2. Выбрать источник угла pitch и **настроить фильтры** (ФНЧ, комплементарный, Калман).
3. Выбрать алгоритм управления (**PD-PI**, **Classic PID**, **SMC**).
4. Записать телеметрию, оценить качество фильтрации и регулятора.
5. Сохранить параметры во Flash.

---

## 2. Схема подключения

```
ПК (Linux) ── USB ──► ESP8266 (/dev/ttyUSB0, 115200)
                          │
                          └── UART ──► ESP32 (на борту UAV)
```
---

## 3. Объяснение назначения кнопок на экране

1. Кнопка **«Log on»** — на экране появляются сигналы: угол, угловая скорость, управляющий сигнал.
2. Кнопка **«Log off»** — сигналы (угол, угловая скорость, управляющий сигнал) исчезают с экрана.
3. Поле **«Желаемый угол»** — желаемый угол поворота квадрокоптера.
4. Кнопка **«Dump Log»** — запись переходного процесса во время работы системы и сохранение его в отдельный файл на ПК. Порядок использования:
  - Сначала задать угол, на который должен повернуться квадрокоптер.
  - После ввода, например, 20 градусов нажать **Enter** и наблюдать, как квадрокоптер поворачивается на 20 градусов.
  - Затем подождать 6–7 секунд, чтобы данные записались в память. После этого создаётся файл `log/takelog_YYYYMMDD_HHMMSS.csv`, который сохраняется в папке **log** (имя файла соответствует времени записи). Пользователю нужно изменить название последнего файла в файле `flix.pg` и с помощью Gnuplot построить график лога (в терминале команда: `gnuplot flix.pg` — результат появится на экране).

## 4. Порядок запуска (каждый сеанс)

Выполняйте шаги **строго по порядку**.

| Элемент на экране | Назначение |
| ----------------- | ---------- |
| **Connected** (зелёный) | USB открыт **и** ESP32 отвечает на команды (раз в 2 с шлётся `sys`). |
| **Not connected** (красный) | Нет порта или борт молчит более 5 с. |
| **START** | Команда `arm`. Кнопка становится красной, статус — **ARMED**. |
| **STOP** | Команда `disarm`. Тяга сбрасывается в 0, статус — **STOPPED** (зелёный). |
| **Thrust** + **0%** | Общая тяга четырёх моторов (`mtr 4`). Перед **START** должно быть **0%**. |
| **Save Parameter** | Записать все текущие параметры во Flash (`save`). |

**Важно:** **START** блокируется, если тяга > 0. Сначала **STOP** и **0%**, потом **START**.

### 4.2 Канал pitch (режим стенда)

| Элемент | Назначение |
| ------- | ---------- |
| **Pitch on** | Включить момент по pitch (`dscnl 2 0`). По умолчанию включён. |
| **Disable** | Отключить все каналы автоматического управления (`dscnl 4 1`) и перейти в **ручной режим**. |

На стенде roll и yaw **не выводятся отдельными переключателями** — при запуске Qt автоматически отключает roll/yaw и оставляет только pitch.

### 4.3 Задание угла

| Элемент | Назначение |
| ------- | ---------- |
| **Желаемый угол** | Желаемый угол pitch в **градусах**. После ввода нажмите **Enter** → команда `p SetPt` (внутри переводится в millirad). |

Пример: введите `20`, нажмите **Enter** — коптер должен стремиться к 20° по pitch.

### 4.4 Регулятор (Control mode + вкладки)

Внизу слева — блок **Control mode** и три вкладки:

| Кнопка / вкладка | `CtrlAlg` | Описание |
| ---------------- | --------- | -------- |
| **PD-PI** | 0 | Двухконтурный PD-PI (`KrenCtrl`). |
| **PID** | 1 | Каскадный PID (**Classic PID** на вкладке). |
| **SMC** | 2 | Скользящий режим (Sliding Mode). |

Активный режим подсвечивается **синим**. Выбор сохраняется между сеансами.

**Менять режим только в состоянии STOP (disarm).** После смены нажмите **Save Parameter**.

Кнопка **Set Default** (под вкладками) — восстановить безопасные коэффициенты PD-PI / PID / SMC и записать их во Flash.

#### Вкладка **Classic PID** (основная для стенда)

| Поле на экране | Параметр Flash | Внешний контур (угол θ) |
| -------------- | -------------- | ----------------------- |
| Pitch θ Kp | `PITCH_P` | пропорциональный |
| Pitch θ Ki | `PITCH_I` | интегральный |
| Pitch θ Kd | `PITCH_D` | дифференциальный |

| Поле на экране | Параметр Flash | Внутренний контур (ω) |
| -------------- | -------------- | --------------------- |
| Pitch ω Kp | `PRateP` | пропорциональный |
| Pitch ω Ki | `PRateI` | интегральный |
| Pitch ω Kd | `PRateD` | дифференциальный |

После изменения — **Enter** в поле (отправка на борт). Для постоянного хранения — **Save Parameter**.

Значения по умолчанию (кнопка **Set Default**): θ — 2.5 / 0 / 0; ω — 0.18 / 0.025 / 0.005.

#### Вкладка **PD-PI**

Параметры roll (префикс **R**) и pitch (префикс **P**):

| Метка на экране | Flash | Смысл |
| --------------- | ----- | ----- |
| RKpf, RKpv, RKdv | `RKPF`, `RKPV`, `RKDV` | Roll — коэффициенты PD |
| RTv, RTmu, RTi, RTe, RTm | `RTV`, `RTmu`, `RTI`, `RTE`, `RTM` | Roll — время/инерция |
| PKpf, PKpv, PKdv | `PKPF`, `PKPV`, `PKDV` | Pitch — коэффициенты PD |
| PTv, PTmu, PTi, PTe, PTm | `PTV`, `PTmu`, `PTI`, `PTE`, `PTM` | Pitch — время/инерция |
| Kal Kf rol / pit | `rKlmf`, `pKlmf` | EMA фильтр угла PD-PI |
| Kal Kv rol / pit | `rKlmv`, `pKlmv` | EMA фильтр скорости PD-PI |

#### Вкладка **SMC**

| Поле | Flash | Смысл |
| ---- | ----- | ----- |
| λ (SMCLm) | `SMCLm` | Наклон скользящей поверхности |
| K (SMCK) | `SMCK` | Усиление в `tanh` |
| Φ (SMCPhi) | `SMCPhi` | Толщина пограничного слоя |
| K_rate (SMCKrt) | `SMCKrt` | Вес угловой скорости в `s` |
| Ki (SMCKi) | `SMCKi` | Интеграл по ошибке |
| I_max (SMCImax) | `SMCImax` | Ограничение интеграла |
| V_max (SMCVmax) | `SMCVmax` | Ограничение `λ·e` |
| LPF α / τ | `SMCLpfa`, `SMCLpft` | Сглаживание выхода SMC |

### 4.5 Источник pitch и фильтры IMU

**Pitch source**

| Режим | Параметр | Поведение |
| ----- | -------- | --------- |
| **Quaternion** | `QuatEn=1` | Угол из фильтра Mahony. |
| **No Quaternion** | `QuatEn=0` | Угол из acc / complementary / Kalman. |

**IMU Filter** (группа справа)

| Режим | `LPF_En` | Когда доступен |
| ----- | -------- | -------------- |
| **Use LPF** | 1 | Только с **Quaternion** |
| **No LPF** | 0 | Quaternion или No Quaternion (acc) |
| **Complementary** | 2 | Только **No Quaternion** |
| **Kalman** | 3 | Только **No Quaternion** |

Дополнительные поля:

| Поле | Flash | Назначение |
| ---- | ----- | ---------- |
| **α LPF** | `LPFAlp` | ФНЧ для acc, rates и D-составляющей PID |
| **α (gyro)** | `CFAlpha` | Комплементарный фильтр (по умолчанию 0.98) |
| **Q angle** | `KQang` | Шум процесса Калмана |
| **R meas** | `KRmeas` | Шум измерения acc в Калмане |

> При **Complementary** или **Kalman** ФНЧ acc/rates **всегда включён** (`LPF_En ≥ 0.5`).

### 4.6 Графики и запись данных

| Элемент | Назначение |
| ------- | ---------- |
| **Log on** | Включить поток телеметрии (`log 1`) — графики pitch, ω, момент. |
| **Log off** | Выключить поток (`log 0`). |
| **Pitch θ (rad)** | Показать/скрыть угол на графике. |
| **Pitch ω rad/s** | Показать/скрыть угловую скорость. |
| **u pitch ×1000** | Показать/скрыть управляющий момент. |
| **Dump Log** | Считать буфер ~6000 отсчётов с борта (`takelog`) и сохранить CSV на ПК. |
| **Get parameters** | Команда `p` — вывести все параметры Flash в текстовое поле лога. |

На графике (легенда):

- **pitch θ (°)** — угол pitch;
- **pitch ω (°/s)** — угловая скорость;
- **Torque pitch** — момент по pitch.

3D-модель справа поворачивается по телеметрии pitch (файл `flixESPDrone/uav.stl`).

### 4.7 Ручной режим (Manual mode)

Активируется кнопкой **Disable**:

1. Надпись **Manual mode selected**.
2. Снимается **Pitch on**.
3. Доступны ползунки **Front** и **Rear** для прямого управления парами моторов (`mfront`, `mrear`).

Чтобы вернуться к автоматическому pitch: снова включите **Pitch on** (ручной режим выключится).

## 5. Порядок работы (каждый сеанс)

Выполняйте шаги **строго по порядку**.

### Шаг 0 — Внешняя проверка

Проверьте крепление пропеллеров, провода питания и надёжность крепления на оси стенда.

### Шаг 1 — Питание и USB

1. Удерживайте квадрокоптер **горизонтально** и подайте питание 1–2 с (калибровка гироскопа).
2. Подключите ESP8266 к ПК по USB.
3. Убедитесь: `ls /dev/ttyUSB0` (или ваш порт). Не держите открытым Arduino Serial Monitor — он занимает порт.

### Шаг 2 — Запуск Qt

```bash
cd Qt_file && ./Qt_Flix
```

Дождитесь **Connected** (зелёный). Если **Not connected** — проверьте питание ESP32, кабель и порт.

### Шаг 3 — Безопасная готовность

| Проверка | Ожидание |
| -------- | -------- |
| Статус | **STOPPED** (зелёный) |
| Thrust | **0%** |
| Канал | **Pitch on** включён |
| Connected | зелёный |

При необходимости нажмите **Get parameters** — в логе появятся параметры Flash.

### Шаг 4 — Выбор регулятора и фильтра

1. Убедитесь, что система **disarm** (**STOP**).
2. Выберите **Control mode**: для первого опыта — **PID** (вкладка **Classic PID**).
3. Выберите **Pitch source** и **IMU Filter** (см. раздел 7).
4. При изменении коэффициентов — **Enter** в поле, затем **Save Parameter**.

### Шаг 5 — Эксперимент с автоматическим pitch

1. Удерживайте квадрокоптер в исходном положении.
2. Нажмите **START** (**ARMED**).
3. Плавно увеличьте **Thrust** до **20%**.
4. Опустите руки — коптер должен удерживать баланс по pitch.
5. Убедитесь, что график обновляется (**Log on** включается автоматически при connect/arm).
6. В поле **Желаемый угол** введите угол (например `15`), нажмите **Enter**.
7. При неустойчивости — подстройте **Classic PID** (сначала ω-контур, затем θ).

### Шаг 6 — Запись переходного процесса (Dump Log)

1. Задайте **Желаемый угол** (например `20`), **Enter**.
2. Дождитесь установления (или наблюдайте переход).
3. Убедитесь, что график «живой» не менее **6 с** после connect/arm (буфер на борту заполняется в фоне).
4. Нажмите **Dump Log**. Кнопка покажет **Receiving...**, затем в логе — путь к файлу.

Файл сохраняется в:

```
~/Documents/UAV_Projects/project_quad/UAV_Lon_ESP8266/log/takelog_YYYYMMDD_HHMMSS.csv
```

Колонки: `pitch_deg`, `gyro_dps`, `torque`.

**Построение графика в Gnuplot** (пример `log/flix.pg`):

```gnuplot
datafile = "takelog_YYYYMMDD_HHMMSS.csv"
set datafile separator ","
set terminal pngcairo size 1200,700
set output "takelog_YYYYMMDD_HHMMSS.png"
set grid
set xlabel "Sample"
set ylabel "Value"
plot datafile using 0:1 with lines title "Pitch (deg)", \
     datafile using 0:2 with lines title "Gyro (deg/s)", \
     datafile using 0:3 with lines title "Torque"
```

Запуск: `cd log && gnuplot flix.pg`

### Шаг 7 — Остановка и сохранение

1. **STOP** — disarm, тяга 0, setpoint pitch сброшен.
2. **Save Parameter** — если меняли коэффициенты и хотите сохранить во Flash.

## 6. Настройка фильтров

### 6.1 ФНЧ (`LPFAlp`)

**Цель:** убрать ВЧ-шум gyro, не добавляя большой задержки.

| Симптом | Действие |
| ------- | -------- |
| Шум на графике ω | **Уменьшить** `LPFAlp` (0.2 → 0.1 → 0.05) |
| Вялый отклик | **Увеличить** `LPFAlp` (0.2 → 0.3) |

### 6.2 Комплементарный фильтр (`CFAlpha`)

**Цель:** баланс gyro (быстро, дрейф) и acc (медленно, шум при вибрации).

| α (`CFAlpha`) | Поведение |
| ------------- | --------- |
| Близко к **1.0** (0.99) | Больше доверия gyro |
| **0.95–0.98** | Умеренная коррекция по acc |
| **< 0.95** | Сильная привязка к acc — риск колебаний |

**Порядок:** **No Quaternion** → **Complementary** → попробовать 0.995, 0.98, 0.95 → наблюдать график и поведение на стенде.

Формула: `pitch_comp = α·(pitch_comp + rates.y·dt) + (1−α)·pitch_acc`

### 6.3 Фильтр Калмана (`KQang`, `KRmeas`)

| Параметр | Интуиция |
| -------- | -------- |
| **Q angle** (`KQang`) | ↑ Q — быстрее следование gyro; ↓ Q — более гладкий угол |
| **R meas** (`KRmeas`) | ↑ R — меньше влияние acc; ↓ R — сильнее подтягивание к acc |

**Старт:** `KQang=0.001`, `KRmeas=0.03`.

**Порядок:**

1. **No Quaternion** → **Kalman**.
2. Зафиксировать `KRmeas=0.03`, менять `KQang`: 0.0005 → 0.001 → 0.005.
3. Зафиксировать лучший Q, менять `KRmeas`: 0.01 → 0.03 → 0.1.
4. Сравнить с Complementary (команда `pit` в serial-терминале).
5. **Save Parameter**.

## 7. Краткая шпаргалка по режимам pitch

| Quaternion | IMU Filter | Источник угла для регулятора |
| ---------- | ---------- | ---------------------------- |
| вкл | Use LPF / No LPF | Mahony quaternion |
| выкл | No LPF | Сырой acc (`pitch_acc`) |
| выкл | Complementary | `pitch_comp` |
| выкл | Kalman | `pitch_kalman` |

По умолчанию при первом запуске Qt обычно **Quaternion** + **Use LPF**.

## 8. Частые проблемы

| Проблема | Решение |
| -------- | ------- |
| **Not connected** | Питание ESP32, кабель USB, порт не занят другой программой |
| Не могу нажать **START** | Thrust должен быть **0%** |
| **Dump Log**: 0 samples | Подождите ≥6 с с живым графиком, затем повторите |
| График пустой | **Log on** или переподключите (arm включает log) |
| Коптер не реагирует на угол | Проверьте **Pitch on**, не включён ли **Disable** |
| Смена PID/SMC не помогает | Меняйте **CtrlAlg** только при **STOP**, затем **Save Parameter** |
