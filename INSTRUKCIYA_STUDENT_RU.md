# Пошаговая инструкция для студентов

## Стенд UAV Lon ESP8266 — запуск, настройка, фильтры и измерения

---

## 1. Цель стенда

Квадрокоптер закреплён на **неподвижной оси** (режим **1 степень свободы** — в основном **pitch**).  
Задачи лабораторной работы:

1. Запустить наземную станцию (Qt) и связь с бортом (ESP32 через ESP8266).
2. Выбрать источник угла pitch и **настроить фильтры** (ФНЧ, комплементарный, Калман).
3. Выбрать алгоритм управления (**PD-PI**, **каскадный PID**, **SMC**).
4. Записать телеметрию, оценить качество фильтрации и регулятора.
5. Сохранить параметры во Flash.

---

## 2. Схема подключения

```
ПК (Linux) ── USB ──► ESP8266 (/dev/ttyUSB0, 115200)
                          │
                          └── UART ──► ESP32 (на борту UAV)
```

## Объяснение назначения кнопок на экране

1. Кнопка **«START»** — запуск процессов и управления системой (после нажатия появляется красный фон со словом **ARMED**).
2. Кнопка **«STOP»** — остановка работы системы (после нажатия появляется зелёный фон со словом **STOPPED**).
3. Кнопка **«Save Parameter»** — сохраняет все подобранные параметры системы во Flash-память ESP32.
4. Кнопка **«Log on»** — на экране появляются сигналы: угол, угловая скорость, управляющий сигнал.
5. Кнопка **«Log off»** — сигналы (угол, угловая скорость, управляющий сигнал) исчезают с экрана.
6. Поле **«Угол»** — желаемый угол поворота квадрокоптера.
7. Кнопка **«Take Log»** — запись переходного процесса во время работы системы и сохранение его в отдельный файл на ПК. Порядок использования:
  - Сначала задать угол, на который должен повернуться квадрокоптер.
  - После ввода, например, 20 градусов нажать **Enter** и наблюдать, как квадрокоптер поворачивается на 20 градусов.
  - Затем подождать 6–7 секунд, чтобы данные записались в память. После этого создаётся файл `log/takelog_YYYYMMDD_HHMMSS.csv`, который сохраняется в папке **log** (имя файла соответствует времени записи). Пользователю нужно изменить название последнего файла в файле `flix.pg` и с помощью Gnuplot построить график лога (в терминале команда: `gnuplot flix.pg` — результат появится на экране).

## 5. Порядок запуска (каждый сеанс)

Выполняйте шаги **строго по порядку**.

### Шаг 0 — Внешняя проверка

Провести внешнюю проверку; особенно важно, чтобы крепление пропеллеров было плотным, а также проверить состояние проводов питания.

### Шаг 1 — Питание и USB

1. Держите квадрокоптер на **горизонтальной** поверхности **параллельно** земле и **подайте** питание. Во время подачи питания плотно удерживайте квадрокоптер 1–2 секунды — идёт калибровка гироскопа.
2. Подключите ESP8266 к ПК через разъём USB.
3. Убедитесь, что порт доступен: `ls /dev/ttyUSB0` (или ваш порт). Не запускайте другие программы (например, Arduino IDE), так как они занимают порт.

### Шаг 2 — Запуск Qt

### Шаг 3 — Проверка связи

В нижней области лога (или через serial-терминал) должны появляться ответы на команды.  
Команда `**p`** (кнопка **Get parameters**) выводит все параметры Flash.

### Шаг 4 — Безопасная готовность


| Проверка      | Действие                                           |
| ------------- | -------------------------------------------------- |
| Состояние     | **STOP** (**STOPPED**) отображается зелёным цветом |
| Thrust (тяга) | **0%**                                             |
| Канал pitch   | **Pitch on** (по умолчанию)                        |


Если нажать кнопку **«Disable All»**, включается режим ручного управления (на экране появляется **«Manual mode selected»**).  
Порядок действий в ручном режиме:

1. Нажать кнопку **«Disable All»**.
2. Нажать кнопку **«START»**.
3. Увеличить ползунок **Thrust**.
4. Чтобы увеличить/уменьшить тягу двух передних моторов — двигать ползунок **Front**.
5. Чтобы увеличить/уменьшить тягу двух задних моторов — двигать ползунок **Rear**.

### Шаг 5 — Выбор регулятора

Регулятор выбирается нажатием кнопки **PD-PI** / **PID** / **SMC** (внизу окна, **Control mode**).  
По умолчанию — **PID** (синий фон).  
*(Примечание: PD-PI пока не подключён и не работает.)*

### Шаг 6 — Подбор параметров регуляторов

Есть 3 вкладки (**PD-PI**, **PID**, **SMC**), в которых нужно изменять параметры регуляторов.  
Параметры **PID**-регулятора:

- **Pitch θ Kp** — пропорциональный коэффициент по углу.
- **Pitch θ Ki** — интегральный коэффициент по углу.
- **Pitch θ Kd** — дифференциальный коэффициент по углу.
- **Pitch ω Kp** — пропорциональный коэффициент по угловой скорости.
- **Pitch ω Ki** — интегральный коэффициент по угловой скорости.
- **Pitch ω Kd** — дифференциальный коэффициент по угловой скорости.

По умолчанию в Qt уже подобраны параметры PID-регулятора — этого достаточно, чтобы система работала стабильно.

### Шаг 7 — Старт и проведение эксперимента

1. Держите квадрокоптер в исходном положении (параллельно земле) и не опускайте руки.
2. Нажмите кнопку **«START»**.
3. Увеличьте **Thrust** до **20%**.
4. Опустите руки — квадрокоптер должен удерживать баланс.
5. Включите **Log on** для графиков pitch / gyro / момент.
6. Задайте разные углы для проверки поворота квадрокоптера.
7. Если квадрокоптер колеблется и работает неустойчиво — измените параметры PID-регулятора.
8. По умолчанию используется режим **Quaternion**.
9. Можно использовать режим **No Quaternion**.
  При режиме **No Quaternion** по умолчанию всегда используется фильтр низких частот (**LPF**) с коэффициентом α фильтра LPF; дополнительно можно использовать:
  1. **Комплементарный фильтр (Complementary)** с коэффициентом фильтра α (gyro) → параметр Flash `**CFAlpha`** (по умолчанию **0.99**).
    Формула комплементарного фильтра:  
     `pitch_comp = α · (pitch_comp + rates.y·dt) + (1−α) · pitch_acc`
  2. **Фильтр Калмана** с параметрами, которые можно настроить напрямую:

    | Параметр    | Назначение                                                   |
    | ----------- | ------------------------------------------------------------ |
    | **Q angle** | Шум процесса угла (↑ → фильтр «верит» гироскопу)             |
    | **R meas**  | Шум измерения акселерометра (↑ → сильнее сглаживание по acc) |


### Шаг 8 — Сохранение

После удачной настройки: кнопка **Save Parameter** или команда `save` — параметры записываются во Flash ESP32.

## Объяснение фильтров

### 1. Настройка ФНЧ (`LPFAlp`)

**Цель:** убрать высокочастотный шум gyro, не добавляя большой задержки.


| Симптом                      | Действие                                  |
| ---------------------------- | ----------------------------------------- |
| Шум на графике ω             | **Уменьшить** `LPFAlp` (0.2 → 0.1 → 0.05) |
| Запаздывание, «вялый» отклик | **Увеличить** `LPFAlp` (0.2 → 0.3)        |


> При **Complementary/Kalman** (`LPF_En=2/3`) ФНЧ acc/rates **всегда включён** (условие `LPF_En ≥ 0.5`).

### 2. Настройка комплементарного фильтра (`CFAlpha`)

**Цель:** баланс между гироскопом (быстро, дрейф) и акселерометром (медленно, шум при вибрации).


| α (`CFAlpha`)           | Поведение                                               |
| ----------------------- | ------------------------------------------------------- |
| Близко к **1.0** (0.99) | Больше доверия gyro → быстрее, но накапливается дрейф   |
| **0.95–0.98**           | Чуть больше коррекции по acc                            |
| **< 0.95**              | Сильная привязка к acc → возможны колебания на вибрации |


**Порядок для студентов:**

1. **No Quaternion** → **Complementary**.
2. Попробовать **0.995**, **0.98**, **0.95**.
3. После каждого шага наблюдать поведение квадрокоптера и сигналы на экране.

### 3. Настройка фильтра Калмана (`KQang`, `KRmeas`)

**Интуиция:**

- `**KQang`** (Q) — насколько модель «ожидает» изменение угла.  
  - ↑ Q → фильтр быстрее следует gyro.  
  - ↓ Q → более гладкий угол.
- `**KRmeas**` (R) — доверие к акселерометру.  
  - ↑ R → меньше влияние acc (сглаживание).  
  - ↓ R → сильнее подтягивание к `pitch_acc`.

**Рекомендуемый старт:** `KQang=0.001`, `KRmeas=0.03`.

**Порядок:**

1. **No Quaternion** → **Kalman**.
2. Зафиксировать `KRmeas=0.03`, менять `KQang`: 0.0005 → 0.001 → 0.005.
3. Зафиксировать лучший Q, менять `KRmeas`: 0.01 → 0.03 → 0.1.
4. Сравнить с Complementary (`pit`: `pitch_kal` vs `pitch_comp`).
5. `save`.
